- **대시보드 무영향** — ozkids-dashboard(GAS)는 자체 하드코딩 토큰을 쓰므로 위 토큰 교체와 무관.

### ⚠️ 발견된 영상 결함 (해결 방향 확정, 구현 예정)
- **upload_video base64 방식은 영상에 안 됨** — Meta advideos는 영상 base64(JSON source)를 지원 안 함 → 390/1363030 거부. 크기 무관(1.75MB도 실패). 즉 **단건 영상은 한 번도 동작한 적 없음**(1885183에 가려 안 보였던 것). 위 "IMAGE·캐러셀 성공"에 영상은 포함 안 됨.
- **Vercel 4.5MB 본문 한도** — 영상 base64가 본문을 키워 ~4.4MB↑에서 엣지가 413 FUNCTION_PAYLOAD_TOO_LARGE로 함수 실행 전 차단. 이 413엔 CORS 헤더가 없어 브라우저엔 "Failed to fetch"로 보임. 3MB 영상이 base64 ~4MB 되어 여기 걸림.
- **해결 방향(확정): `file_url` 방식** — advideos에 base64 대신 Drive 공개 다운로드 URL을 `file_url`로 넘겨 Meta가 서버측에서 직접 받게 함. proxy가 영상 바이트를 안 만짐 → Vercel 4.5MB 한도·메모리·타임아웃 전부 회피. **대량(Drive) 경로는 이 방식으로 구현**(3단계).
- **단건 폼 영상은 백로그** — 로컬 PC 파일이라 공개 URL이 없어 file_url 불가. resumable 업로드 등 별도 작업 필요. 원래 동작한 적 없으므로 이번 범위 밖(10번 백로그).

결과: 단건 IMAGE·캐러셀 라이브 생성 성공(검증됨). 영상은 file_url 방식으로 대량 경로에서 새로 구현 예정. Drive 검색·다운로드·파일명 변환·create_ad 추출까지 완료.

남은 임시 진단 액션(정리 대상): `pa_resolve_test`(PA용 유지), `drive_test`(대량 완성 후 제거). **PA 작업까지 끝난 뒤 일괄 정리.**

## 1. 목적과 범위

'광고 세팅' 탭 하단에 **'광고 대량 업로드'** 섹션을 신규 추가한다. 엑셀 시트 + Google Drive 폴더의 미디어로 광고를 한 번에 N개(보통 20개) 생성한다. 기존 단건 폼은 그대로 둔다.

**이번 범위: image / video / carousel (3종).**
- **PA는 이번 범위에서 제외** — 릴스 URL→media id 변환에 Instagram 권한(instagram_basic 등) 보강이 필요한데 아직 미해결. PA는 권한 확인 후 별도로 추가(아래 7번 참고).
- collection도 제외(인스턴트 경험 수작업 필요, 단건 폼 유지).

## 2. 미디어 처리 — Google Drive 폴더 + 파일명 매칭

### 핵심 방식
- 광고 소재 전용 Drive 폴더: **`1Owv3k2aYgeatjnke-pS-PVKvN2AIggP5`** (고정, '링크 있는 모든 사용자-뷰어' 공개).
- 사용자는 이 폴더에 소재를 올리고, 시트에는 **파일명만** 적는다(긴 Drive 링크 복사 안 함).
- proxy가 Drive API(`GOOGLE_DRIVE_API_KEY`)로 폴더에서 파일명 검색(files.list) → file ID 획득.
- **이미지**: files.get?alt=media로 받아 upload_image(base64) → image_hash. (이미지 base64는 작아 문제없음)
- **영상**: Drive 공개 다운로드 URL을 advideos의 **`file_url`** 로 넘김 → Meta가 서버측에서 직접 다운로드 → video_id. **proxy가 영상 바이트를 안 만짐**(Vercel 4.5MB 한도·메모리·타임아웃 회피). base64 방식 금지.

### 파일명 = 광고세트명 = 광고명 (A 방식, 통합)
- `lib/filename.js`의 `parseMediaFilename()` 사용(완료·검증됨).
- 확장자 제거 + 끝 `(숫자)` 카드번호만 제거(중간 `(400)` 등 카테고리번호 보존) → 광고명·광고세트명·소재명.
- 예: `원피스모리본_메인컷화이트_260617.jpg` → `원피스모리본_메인컷화이트_260617`.

### Drive 접근 (확정)
- 공개 폴더 + `GOOGLE_DRIVE_API_KEY`(Vercel 환경변수, 등록 완료). 서비스 계정 안 씀.
- 검색·다운로드 모두 Drive API. 스파이크에서 90MB 영상까지 완전 수신 확인.
- Drive 검색/URL 생성 로직은 `lib/drive.js`로 추출해 drive_test·본구현 공유(권장).

### 매칭 검증
- 폴더에 해당 파일명이 없거나 같은 이름이 둘 이상이면 → 미리보기 단계에서 빨간 행 + 사유 표시(생성 전 차단).
- 동일 FILE은 한 번만 처리하고 hash/id 캐시 재사용.

## 3. 시트 스키마

헤더명 기준 파싱(열 순서 의존 금지). 컬럼:

| 컬럼 | 적용 | 필수 | proxy 매핑 / 동작 |
|---|---|---|---|
| type | 전체 | ✅ | image / video / carousel |
| 광고그룹 | carousel | ✅(carousel) | 같은 값의 연속 행 = 한 캐러셀의 카드들. image/video는 비움 |
| 캠페인 | 전체 | ✅ | campaign_id 또는 캠페인명. 기존 캠페인 재사용(신규 생성 안 함). carousel은 그룹 첫 행에만 |
| 파일명 | image/video, carousel 각 카드 | ✅ | Drive 폴더에서 검색. 확장자 뗀 이름 = 광고세트명·광고명(A 방식). image=이미지(base64 업로드), video=영상(Drive 공개 URL을 advideos file_url로), carousel=카드 이미지 파일((1)(2)(3) 포함) |
| 캡션 | image/video/carousel | ⬜ | link_data.message. 비우면 Gemini 자동작성. carousel은 그룹 첫 행에 1개(광고당 1개) |
| 랜딩URL | image/video | ✅ | link_data.link(+UTM 자동). carousel은 '카드랜딩URL' 사용하므로 비움 |
| 카드랜딩URL | carousel | ✅(carousel) | child_attachments[].link(+UTM). 카드(행)마다 |
| 카드헤드라인 | carousel | ⬜ | child_attachments[].name = 카드 아래 노출되는 제목(소비자 노출). 카드(행)마다. **비우면 name 자체를 생략**(광고세트명 절대 미노출). image/video는 비움 |

### image / video
- 한 행 = 광고 1개. 파일명 + 캠페인 + 랜딩URL 필수. 광고명은 파일명에서 자동.

### carousel (모양 B — 카드마다 행 나누기)
- 같은 **광고그룹** 값의 연속 행 = 한 캐러셀. 카드 순서 = 행 순서(= 파일명의 (1)(2)(3) 순서).
- **각 행에 카드 이미지 파일명을 직접 적는다** (예: `신상카테고리(142);ai코디컷_슬라이드_260420 (1)`, ` (2)`, ` (3)`). `(1)`=1번 카드, `(2)`=2번 카드.
- 카드별 **카드랜딩URL**을 행마다 적는다(카드마다 다른 URL).
- 카드별 **카드헤드라인**(child_attachments[].name)을 행마다 적는다(소비자 노출 제목). 비우면 제목 없이 나감. **광고세트명을 카드 제목에 넣지 않음**(과거 버그 수정됨).
- 캡션은 그룹 첫 행에 1개. 광고세트명/광고명은 공통 부분(번호·확장자 뗀 이름)에서 자동.
- 번호 없는 원본 파일은 올리지 않는다(방식 1). 카드는 (1)(2)(3)만.
- 카드 2~10장. 한 캐러셀 = 광고 1개 = 광고세트 1개.

### 예시 (캐러셀)
| type | 광고그룹 | 파일명 | 캡션 | 카드랜딩URL |
|---|---|---|---|---|
| carousel | A | 신상카테고리(142);ai코디컷_슬라이드_260420 (1) | 신상 모음전 | https://ozkiz.com/p/1 |
| carousel | A | 신상카테고리(142);ai코디컷_슬라이드_260420 (2) | | https://ozkiz.com/p/2 |
| carousel | A | 신상카테고리(142);ai코디컷_슬라이드_260420 (3) | | https://ozkiz.com/p/3 |

→ 광고세트명/광고명 = `신상카테고리(142);ai코디컷_슬라이드_260420` (번호·확장자 제거).

### 검증
- type ∈ {image, video, carousel}. 캠페인 매칭 실패 → 에러. URL 형식 불량 → 에러.
- 파일명이 Drive 폴더에 없음/중복 → 에러(행/카드 단위).
- carousel: 같은 광고그룹 카드 2장 미만 → 에러.

## 5. proxy 신규 엔드포인트: `/api/bulk-create-ads`

기존 create_ad를 **재사용**하는 얇은 오케스트레이션 레이어. 단건 로직 복제 금지.

### 처리
1. 행 검증(3번). carousel은 광고그룹으로 묶어 그룹 단위 검증.
2. 캠페인명 → campaign_id(get_campaigns 캐시).
3. 미디어 준비: Drive 폴더에서 파일명 검색 → 다운로드 → upload_image/video → hash/id. carousel은 카드 수만큼 image_hash 배열. FILE 캐시 재사용.
4. 광고명/광고세트명 = 파일명(확장자·카드번호 제거)으로 자동 생성.
5. 기존 create_ad 흐름으로 광고세트→크리에이티브→광고(PAUSED). carousel은 child_attachments.
6. 행/그룹별 결과 누적 반환(부분 실패 격리).

### 응답
```
{ "summary": {"total","success","failed"},
  "results": [ {"rowIndex","status":"ok|error","stage":"validate|campaign|media|adset|creative|ad","ad_id","adset_id","ad_permalink","message"} ] }
```

### 안정성
- 행 순차 또는 소규모 동시(2~3). Meta rate limit·Drive 대용량 고려.
- 부분 실패 허용. 이미 성공한 행은 재전송 시 제외(rowIndex+광고명 기준).

## 6. 프론트 UI — '광고 대량 업로드' 섹션

위치: '광고 세팅' 탭, 단건 폼 아래 별도 카드.

1. **템플릿 다운로드** — 최신 .xlsx 제공.
2. **시트 업로드** — .xlsx 파싱(헤더명 기준, SheetJS). carousel은 광고그룹으로 묶어 표시.
3. **미리보기** — 행/그룹별 파싱 + 검증. 유효=기본색, 에러=빨간 행+사유. Drive 파일명 존재여부 "확인 중→OK/실패" 뱃지. 자동 생성될 광고세트명 미리보기. 캐러셀은 카드 묶음으로 표시.
4. **공통 세팅 재확인** — 자동 고정값 표시(전환수 극대화 반영).
5. **"N개 일괄 생성(PAUSED)"** — 유효 행만 `/api/bulk-create-ads` 전송. 진행률.
6. **결과 패널** — 행별 성공/실패·사유·광고 링크. 성공 제외 재시도.

기존 단건 폼/공통 세팅 로직 재사용. HTML form 제출 대신 버튼 핸들러. 대량 전송 중 이탈 경고.

### 6-A. 오류 표시 — 셀 단위로 친절하게 (필수)

사용자가 20행씩 올리므로, 어디를 왜 고쳐야 하는지가 명확해야 한다. **생성 전 미리보기**와 **생성 후 결과** 양쪽에서 오류를 행/셀 단위로 보여준다.

**미리보기 단계(생성 전 검증):**
- 문제 있는 행은 빨간색 표시 + 문제가 된 **특정 셀(열)**을 콕 집어 강조.
- 메시지는 "몇 행 / 어느 컬럼 / 무슨 문제 / 어떻게 고칠지"가 드러나게 구체적으로. 예:
  - "9행 '파일명': Drive 폴더에 '원피스모리본_260617.jpg'가 없음 — 폴더 업로드/철자 확인"
  - "10행 '캠페인': '여름신상'이라는 캠페인을 찾을 수 없음 — 정확한 캠페인명/ID 확인"
  - "12행 '랜딩URL': URL 형식 오류"
  - "광고그룹 A: 카드가 1장뿐 — 캐러셀은 2장 이상 필요"
  - "14행 '파일명': 폴더에 같은 이름 파일이 2개 — 이름 구분 필요"
- 상단에 요약: "총 N행 중 M행 오류" + 오류 목록을 한눈에.
- 자동 생성될 광고세트명/광고명을 미리 보여줘 사용자가 결과를 예측 가능하게.
- 시트를 고쳐 다시 올리면 재검증. **유효한 행만** 생성 버튼 활성화.

**결과 패널(생성 후 — Meta가 거부한 경우):**
- 미리보기를 통과해도 Meta API 단계에서 실패할 수 있음(이미지 규격·비율·용량 부적합, 캠페인-유형 비호환 등 사전 검증 불가한 것). 이 경우 결과 패널에서 행별로 `stage`(media/adset/creative/ad) + Meta 원본 사유(fbErr로 펼친 error_user_msg/subcode)를 그대로 보여줘 어디서 왜 막혔는지 명확히.
- 성공 행은 광고 링크, 실패 행은 사유. 실패 행만 골라 재시도.

> 즉 "미리보기에서 1차로 거르고(파일명·캠페인·URL·카드수 등), 생성 중 Meta 거부는 결과 패널에서 2차로 사유 표시"하는 2단 구조. 둘 다 행/셀 단위로 어디를 고칠지 안내.

## 7. PA (이번 범위 밖 — 별도 트랙)

PA를 대량에 추가하려면 먼저 **Instagram 권한 보강**이 필요하다.
- 현재 토큰 스코프엔 IG 콘텐츠 읽기 권한이 없어, 릴스 URL→V2 media id 변환 불가(스파이크에서 1차 확인됨).
- 새 Business 앱 + ads-automation 시스템 사용자에 `instagram_basic`(+필요 시 `instagram_manage_ads`)을 추가하고, IG 계정(ozkiz_official, 17841402167601732)을 시스템 사용자에 할당한 뒤, pa_resolve_test로 릴스 URL→media id 변환이 되는지 재검증.
- 변환이 풀리면: 릴스 URL만 시트에 적고 proxy가 media id로 변환해 게시물 재사용(source_instagram_media_id). 파트너십(브랜디드 콘텐츠) 권한은 계정 레벨에 이미 연결됨.
- 변환이 안 풀리면: 시트에 V2 media id를 직접 받는 우회(단, PA가 한 번에 10+개라 비현실적 → 권한 보강을 우선).
- PA가 풀리면 시트에 `릴스URL` 컬럼 추가, type에 pa 추가.

## 9. 확정 결정 (변경 금지)

- 대상: image / video / carousel (PA·collection 제외).
- 미디어: Drive 폴더(1Owv3k2aYgeatjnke-pS-PVKvN2AIggP5) + 파일명 매칭.
- 파일명 = 광고세트명 = 광고명(확장자·카드번호 제거).
- 성과목표 OFFSITE_CONVERSIONS, 일일예산 10,000 고정, 1광고=1광고세트, PAUSED, 캠페인 재사용.
- 캐러셀: 카드마다 행 나누기(모양 B), 각 행에 카드 파일명 직접, 카드별 URL, 광고그룹으로 묶기, (1)=1번 카드, 번호없는 원본 미사용, end card 자동추가 OFF.

- **미디어 업로드 방식(확정):** 이미지=base64(`adimages` `bytes`). 영상=Drive 공개 URL을 `advideos`의 `file_url`로 넘겨 **Meta가 직접 ingest**(proxy가 바이트 안 만짐 → Vercel 4.5MB 한도·메모리·타임아웃 회피). 영상은 업로드 후 **ready 폴링 → 썸네일(`video_data.image_url`) 필수**(미지정 시 subcode 1443226). 90MB까지 검증됨.
- **대량 처리 한도(확정):** proxy `maxDuration=60s`. 90MB 영상 1건이 ingest+인코딩대기+생성으로 ~50s 소요 → **프론트는 청크로 호출(영상은 요청당 1건, 이미지는 다건 OK)**, 진행률 표시. 부분실패 격리.
- **코드 구조(확정):** 공용 로직은 `lib/`(CommonJS)로 추출해 단건·대량 공유 — `lib/meta.js`(createAd + uploadImage/uploadVideoByUrl/getVideoStatus/getVideoThumbnail + fbErr), `lib/drive.js`(findByName/downloadBase64/mediaUrl), `lib/filename.js`(parseMediaFilename). ⚠️ `.mjs` 금지(Vercel CJS 런타임이 require 시 ERR_REQUIRE_ESM 500) — 반드시 `.js`+module.exports.
